# Github-Project

A React + Vite application for exploring GitHub users and repositories. The app has two main views: a profile viewer for looking up GitHub users and a repository explorer for searching public repositories.

Visit website: https://github-project-phi.vercel.app

## Features

- Search GitHub user profiles by username
- View profile details including avatar, bio, followers, following, public repositories, public gists, company, location, website, and join date
- Display a user's recent public repositories
- Search public GitHub repositories by keyword
- Sort repository search results by stars, forks, or update date
- Change result order between ascending and descending
- Paginate through repository search results
- Responsive dark interface built with Tailwind CSS
- Hover effects, card shadows, and a subtle mouse-follow glow
- Error and empty-state handling for failed or missing searches

## Tech Stack

- React
- Vite
- Tailwind CSS
- React Icons
- GitHub REST API

## Getting Started

### Prerequisites

Install Node.js and npm. This project uses Vite and npm scripts.

### Installation

Clone the repository:

```bash
git clone https://github.com/Vishnu-Charan-KG/Github-Project.git
cd Github-Project
```

Install dependencies:

```bash
npm install
```

Start the development server:

```bash
npm run dev
```

Open the local URL printed in the terminal, usually:

```txt
http://localhost:5173/
```

## How To Use

### Profile Viewer

1. Select the `Profile Viewer` tab.
2. Enter a GitHub username.
3. Submit the search.
4. View the user's profile details and public repositories.

### Repo Explorer

1. Select the `Repo Explorer` tab.
2. Enter a repository search term, such as `react`, `python`, or `machine learning`.
3. Submit the search.
4. Use the filters to change sorting and order.
5. Use pagination to move through result pages.

## GitHub API Notes

The app uses public GitHub REST API endpoints:

- `GET /users/{username}`
- `GET /users/{username}/repos`
- `GET /search/repositories`
- `GET /repos/{owner}/{repo}`

Requests are unauthenticated, so they are subject to GitHub's public API rate limits. If you extend this project for heavier use, consider adding a backend service or a safe token-based approach.

## Deployment Notes

This is a Vite single-page app. For static hosting, deploy the built `dist` folder generated by:

```bash
npm run build
```

If deploying to a subpath, configure Vite's `base` option to match the deployed path.
